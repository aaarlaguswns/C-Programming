
# 1-1 과제제출

1. 용어 조사
  - 니모닉 기호: 0과 1로 된 기계어 명령을 사람이 이해하기 쉬운 ADD, SUB, MOVE 등의 상징적인 기호로 일대일 대응시킨 것입니다.
  - 로더(Loader): 보조기억장치(HDD/SSD)에 있는 실행 파일을 주기억장치(RAM)로 옮겨 CPU가 실행할 수 있도록 준비해 주는 프로그램입니다
  - 부작용(Side Effect): 전역 변수나 전역 함수를 사용할 때, 의도치 않게 다른 코드 영역에 영향을 주어 발생하는 오류 가능성을 의미합니다
2. 알고리즘 설계와 코딩 중 더 중요한 것
    - 알고리즘 설계가 더 중요하다 코딩은 단순히 언어라는 도구로 구현하는 과정이지만, 알고리즘은 문제를 해결하는 본질적인 절차이자 방법이기 때문입니다
3. 클래스와 객체
  - 클래스: 객체를 만들기 위한 '틀' 또는 '설계도'이며, 사용자가 직접 정의하는 자료형입니다.
  - 객체: 클래스라는 틀을 통해 실제로 생성된 '실체(Instance)'를 의미합니다
4. 절차지향 vs 객체지향 프로그래밍
  - 절차지향: 실행하고자 하는 순서대로 명령어를 나열하며, 흐름도(Flowchart)를 설계하는 것이 중심입니다
  - 객체지향: 객체들을 먼저 정의하고, 이 객체들 사이의 상호 관계와 작용을 중심으로 프로그램을 구현합니다
5. 라이브러리란 무엇인가?
  - 프로그램을 만들 때 자주 사용되는 기능(예: cout, cin 등)을 미리 작성하여 모아놓은 코드 집합입니다
6. 라이브러리가 목적 파일형태로 존재하는 이유
  - 라이브러리는 이미 기계어로 번역된 목적 파일 형태로 주어진다
  - 이는 원본 소스코드를 보호할 수 있고 사용자가 매번 라이브러리 코드를 컴파일할 필요가 없어 개발 시간을 단축해 주기 때문입니다
7. 링크 과정이 필요한 이유
  - 컴파일을 통해 만들어진 목적 파일은 그 자체로 실행될 수 없다
  - 내가 쓴 코드와 프로그램에 필요한 라이브러리 파일들을 하나로 연결해야만 최종적인 실행 파일이 완성되기 때문이다
